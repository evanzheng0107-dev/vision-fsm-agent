# Release Notes — v0.1.0

**Release date**: 2026-06-29
**Codename**: Initial public release

---

## Highlights

- **Rebranded as Vision FSM Agent** — a general-purpose computer-vision +
  finite-state-machine + human-in-the-loop framework for local, controlled
  demos, research, and education.
- **Reproducible synthetic demo** — `demo_app/visual_grid_world.py`
  renders a grid world with OpenCV/numpy. No game, screen capture, or
  external software required. Runs headless and is deterministic given a
  fixed seed.
- **Modular architecture** — 6 independent core modules:
  - `src/fsm.py` — generic FSM engine (guards, actions, callbacks, history)
  - `src/vision.py` — multi-scale OpenCV template matching
  - `src/agent.py` — local rule-based + optional LLM-backed decision agent
  - `src/hil_client.py` / `src/hil_server.py` — human-in-the-loop correction
  - `src/main.py` — environment-agnostic agent loop
- **Safety boundaries** — default environment is synthetic; live
  screen-capture mode is opt-in with PyAutoGUI failsafe enabled. No
  anti-detection or evasion features.
- **50 pytest tests** covering FSM, vision, HIL, config, agent, and
  end-to-end demo execution.
- **GitHub Actions CI** — multi-OS (Ubuntu + Windows) × multi-Python
  (3.9 / 3.11 / 3.13) matrix with coverage reporting.
- **Complete OSS governance** — MIT License, CONTRIBUTING,
  CODE_OF_CONDUCT, SECURITY, AGENTS, CHANGELOG, issue templates, PR
  template with safety checklist, CODEOWNERS.

## Safety

- Intended for **local, controlled demo environments, research, and
  education**.
- **Not** intended for cheating, botting, bypassing platform rules, or
  automating third-party systems without permission.
- No anti-detection, human-mimicry, or evasion features are included.
- The optional live mode requires explicit configuration and keeps
  PyAutoGUI's failsafe enabled.

## Breaking Changes

This is the initial release. There are no prior stable releases to
break compatibility with.

## Known Limitations

- The runtime agent-decision audit log (planned for v0.2.0) is not yet
  implemented.
- FSM topologies are defined in code, not configurable via YAML (planned
  for v0.2.0).
- No web dashboard for HIL corrections (planned for v0.2.0).
- Coverage is ~64%; target is 80%+ by v0.2.0.
